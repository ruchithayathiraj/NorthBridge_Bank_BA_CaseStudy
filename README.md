## 📌 Project Overview

This is a **self-initiated Business Analyst case study** based on a realistic fictional banking scenario. It demonstrates end-to-end BA skills — from requirements gathering and process mapping through to user stories and UAT planning.

> **Disclaimer:** NorthBridge Bank is a fictional organization created for portfolio purposes. All names, data, and scenarios are fictional. The methodology, documentation, and analytical thinking reflect real BA practices.

---

## 🎯 The Business Problem

NorthBridge Bank processes retail loan applications through a **manual, paper-based workflow**. Customers must visit a branch in person, wait an average of **10 business days** for a decision, and have no visibility into their application status. This results in:

- ❌ Low customer satisfaction (CSAT: 58/100)
- ❌ ~12% manual data entry error rate
- ❌ Maximum capacity of 200 applications/month
- ❌ No audit trail for compliance

---

## 💡 The Proposed Solution

Design and define requirements for a **Digital Loan Application System (DLAS)** — a secure web-based platform enabling customers to apply for loans online and receive decisions within **2–3 business days**.

**Expected outcomes:**
- ✅ Processing time reduced from 10 days → under 3 days
- ✅ CSAT score improved from 58 → 80+/100
- ✅ Error rate reduced from 12% → under 2%
- ✅ Capacity increased from 200 → 500+ applications/month

## 📊 User Stories Summary

| Story ID | User Story | Priority |
|---|---|---|
| US-001 | As a customer, I want to apply for a loan online, so that I can get a decision without visiting a branch | Must Have |
| US-002 | As a customer, I want to upload my supporting documents online, so that I don't have to visit the branch with physical papers | Must Have |
| US-003 | As a customer, I want to access my loan application status online, so that I don't have to visit the branch and can get updates anytime | Must Have |
| US-004 | As a loan officer, I want to view all submitted applications in one dashboard, so that I can review and make decisions efficiently | Must Have |
| US-005 | As a loan officer, I want to approve or decline a loan application with a reason, so that the customer understands the decision | Must Have |
| US-006 | As a loan officer, I want the system to automatically retrieve the customer's credit score, so that I don't have to call Equifax manually | Must Have |

## 🔗 Traceability Summary

Every requirement in this project traces back to a documented business problem:

```
Business Pain Point (AS-IS)
        ↓
Business Objective (Section 2)
        ↓
Functional Requirement (Section 7)
        ↓
User Story (US-001 to US-006)
        ↓
Acceptance Criteria (AC-001 to AC-006)
        ↓
UAT Test Case (TC-001 to TC-016)
```

This end-to-end traceability ensures nothing is built without a business reason, and every business need is tested before go-live.
